Description:

Specify the details for launching an instance. Also please make note you need a valid Key and a Security Group for your Virtual Machine:

  • Server Name: You need to give a name to your virtual machine. This name must be unique and should identify the server from the other machines.
  •  
  • User Data: (For advanced users) Pass information as metadata to your virtual machine. Useful if you want to configure your servers at boot time.
  •  
  • Flavor: Server type assigned to your virtual image based on the Virtual CPUs, Memory and Storage needed (Keep in mind that you can add extra storage at any time adding volumes).
  •  
  • Key Name: User Key to control the access to the Server. You have to check that your virtual image can use Key injection
  •  
  • Security Groups: Firewall rules to allow or deny access to the ports of your private network. You should use Security Groups combined with Floating IPs to allow traffic from Internet to your servers.
